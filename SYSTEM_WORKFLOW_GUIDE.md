# 系统工作流功能说明

## 功能概述

为新建项目自动生成系统级工作流，展示项目内容如何转换为视频。系统工作流是只读的，用户只能查看工作原理，无法修改节点和连接。

## 核心特性

### 1. 自动生成系统工作流

当创建新项目时，系统自动：
- 生成一个完整的短剧创作工作流
- 将项目内容（脚本、描述）填充到节点参数中
- 保存到项目的元数据中
- 标记为系统工作流（system: true, readonly: true）

### 2. 系统工作流结构

```
文本输入 → 文生人物 → 人物三视图
         ↘ 文生图 → 图生视频
```

**节点说明**：

1. **文本输入**
   - 自动填充项目源内容（脚本/小说）
   - 显示项目前 500 个字符
   - 标记为「项目脚本」

2. **文生人物**
   - 自动填充人物描述
   - 使用项目风格设置
   - 标记为「角色生成」

3. **文生图**
   - 自动填充场景描述
   - 使用项目名称和风格
   - 标记为「场景生成」

4. **人物三视图**
   - 接收人物形象输入
   - 生成正、侧、背三个视图
   - 标记为「角色三视图」

5. **图生视频**
   - 接收场景图片输入
   - 生成视频片段
   - 标记为「视频生成」

### 3. 只读模式

系统工作流在只读模式下：

- ✅ 可以查看节点和连接
- ✅ 可以查看节点参数
- ✅ 可以执行工作流
- ✅ 可以缩放和平移画布
- ❌ 不能添加、删除节点
- ❌ 不能移动节点
- ❌ 不能连接或断开连接
- ❌ 不能修改节点参数
- ❌ 不能保存工作流

### 4. 界面提示

**左侧节点库**：
- 显示黄色警告框
- 提示「系统工作流 - 这是系统自动生成的工作流，展示项目内容如何转换为视频。您可以查看工作原理，但无法修改。」
- 节点列表半透明，无法拖拽

**工具栏**：
- 显示「系统工作流（只读）」标签
- 保存按钮禁用
- 执行按钮可用（可以查看工作流执行结果）

**节点**：
- 删除按钮隐藏
- 无法拖拽移动
- 可以点击查看参数

## 使用方式

### 查看系统工作流

1. 创建新项目
2. 进入项目的「工作流」标签页
3. 默认显示系统工作流
4. 查看工作流结构和节点参数

### 执行系统工作流

1. 在系统工作流页面点击「执行工作流」
2. 查看每个节点的执行结果
3. 查看生成的图片、视频等内容

### 创建自定义工作流

1. 在工作流页面点击「新建工作流」
2. 从节点库拖拽节点到画布
3. 配置节点参数
4. 连接节点
5. 保存自定义工作流

## 技术实现

### 1. 系统工作流生成

**文件**：`src/lib/workflow/system-workflow.ts`

```typescript
export function createSystemWorkflow(
  projectId: string,
  projectName: string,
  sourceContent: string,
  style?: string
): Workflow {
  // 生成系统工作流
  // 填充项目内容到节点参数
  // 标记为只读
}
```

### 2. 项目创建时自动生成

**文件**：`src/app/api/projects/route.ts`

在项目创建成功后调用：

```typescript
createSystemWorkflowForProject(projectId, projectName, sourceContent, style)
```

### 3. 工作流类型扩展

**文件**：`src/lib/workflow/types.ts`

```typescript
export interface Workflow {
  // ...
  system?: boolean  // 是否为系统工作流
  readonly?: boolean  // 是否只读
}
```

### 4. 前端只读模式

**文件**：`src/components/workflow/workflow-editor-v2.tsx`

```typescript
interface WorkflowEditorProps {
  readonly?: boolean
  isSystem?: boolean
}
```

根据 `readonly` 和 `isSystem` 属性：
- 禁用节点拖拽
- 隐藏删除按钮
- 显示警告提示
- 禁用保存按钮

### 5. 工作流页面

**文件**：`src/app/projects/[id]/workflow/page.tsx`

支持两种模式：
- **系统工作流**：只读模式，展示自动生成的工作流
- **自定义工作流**：可编辑模式，用户可以创建和修改工作流

## 数据流

```
1. 用户创建项目
   ↓
2. 项目保存到数据库/内存
   ↓
3. createSystemWorkflowForProject() 被调用
   ↓
4. 生成系统工作流配置
   ↓
5. 保存到项目元数据
   ↓
6. 用户访问工作流页面
   ↓
7. 加载系统工作流配置
   ↓
8. 渲染只读工作流编辑器
```
