# 剧本撰写工作流程

> **所属 Skill**: gen-video
> **核心用途**: 规范剧本创作意图识别与流程路由
> **何时读取**: 用户需要创作剧本、短剧、漫剧脚本时
> **前置确认**: 用户意图是"撰写剧本"而非"视频生成"——后者直接进 V-Flow

---

## 🎯 一、意图分类与路由决策

**核心判断依据**：用户是否已有完整剧本，以及内容形态的复杂度。

| 意图类型 | 触发特征 | 执行流转路线 |
| :--- | :--- | :--- |
| **已有剧本** | 用户直接上传了完整剧本或极度详尽的逐集大纲 | 告知用户跳过撰写，**直接进入 V1 视频生成** |
| **轻量脚本** | 单条短视频 (1-5分钟)，如产品宣传、解说、品牌故事 | 执行 **S1 → S3(简) → S7** |
| **短剧剧本** | 创作多集的"短剧"、"漫剧"、"连续剧" | 执行 **S1 至 S7** 完整标准流程 |

---

## 📋 二、轻量脚本简化流程 (S1 → S3(简) → S7)

**适用场景**：单条短视频，时长 1-5 分钟，信息密度相对集中。

### S1：需求解析与信息提取

从用户输入中提取以下关键信息：

| 信息维度 | 提取目标 | 示例 |
|---------|---------|------|
| **核心卖点/知识点** | 这条视频要传达什么核心价值？ | 产品功能、品牌理念、知识干货 |
| **目标受众** | 谁会看这条视频？ | 25-35岁职场女性、科技爱好者 |
| **风格偏好** | 什么样的视觉/叙事风格？ | 温暖治愈、科技感、幽默风趣 |
| **行动号召** | 希望观众看完做什么？ | 点击购买、关注账号、转发分享 |

**输出格式**：
```
【需求解析】
核心内容：[一句话概括]
目标受众：[画像描述]
风格定位：[风格关键词]
预期时长：[X分钟]
行动号召：[期望行为]
```

---

### S3(简)：视频脚本大纲（简化版）

**核心目标**：输出一份轻量级的结构化大纲，快速对齐用户意图。

**输出模板**：

```markdown
# 《视频标题》脚本大纲

## 基本信息
- 类型：[产品宣传 / 品牌故事 / 知识解说 / 其他]
- 时长：约 [X] 分钟
- 风格：[关键词]

## 结构设计
### 开场（[X]秒）
- 画面：[简要描述]
- 信息点：[核心钩子]

### 主体（[X]秒）
- 段落1：[画面描述] + [信息点]
- 段落2：[画面描述] + [信息点]
- ...

### 收尾（[X]秒）
- 画面：[简要描述]
- 信息点：[行动号召/升华]

## 关键元素
- 场景：[列出需要的场景]
- 角色：[列出出镜角色/旁白]
- 关键道具/产品：[列出重点展示的物品]

---
**请确认以上大纲是否符合您的预期？确认后我将撰写完整脚本。**
```

**⚠️ 轻量脚本跳过 S4-S6**：简化版大纲确认后，直接进入 S7 撰写完整脚本。

---

## 🎬 三、短剧剧本标准流程 (S1 → S2 → S3 → S4 → S5 → S6 → S7)

**适用场景**：多集连续剧、漫剧、剧情类短剧，需要完整世界观和人物设定。

### S1：输入解析，读取设定

**核心任务**：全面理解用户需求，建立创作框架。

**必读参考文档**（按需）：
- `gen-video/references/drama-script-workflow.md`（完整流程）
- `gen-video/references/scene-rules/`（场景类型规则）

**信息提取清单**：

| 维度 | 内容 | 备注 |
|-----|------|------|
| 剧名 | 原始标题 | 后续文件命名基准 |
| 集数 | 计划集数 | 决定内容密度分配 |
| 单集时长 | 预期每集时长 | 影响场景数量 |
| 类型定位 | 短剧/漫剧/连载 | 决定叙事风格 |
| 核心梗概 | 一句话故事线 | 整体方向锚定 |
| 目标受众 | 画像描述 | 影响台词风格和节奏 |
| 参考作品 | 对标剧名 | 风格参考 |

---

### S2：爆款对标与题材定位（内部思考阶段）

**核心任务**：分析同类爆款，提炼成功要素，但不直接告知用户。

**思考框架**：
1. 同类题材爆款有哪些？（搜索验证）
2. 它们的核心爽点是什么？（情绪钩子）
3. 常见的结构套路是什么？（开场、转折、高潮、收尾）
4. 用户需求与爆款套路的契合点在哪？

**输出形式**：内部 memo，不发给用户。

---

### S3：基础信息梳理 [🛑 强制暂停点]

**核心任务**：向用户呈现完整的世界观、人物设定和分集大纲，等待确认。

**输出模板**：

```markdown
# 《剧名》创作方案

## 一、世界观设定
[时代背景/空间设定/特殊设定]

## 二、核心人物（主角）
### 人物A
- 姓名：[名字]
- 年龄：[年龄段]
- 性格：[核心性格特质]
- 外貌：[关键视觉特征]
- 核心动机：[想要什么]
- 人物弧光：[如何变化]

### 人物B
[同上格式]

## 三、配角阵容
[配角姓名 + 功能定位 + 与主角关系]

## 四、分集大纲
### 第1集：[集名]
- 核心情节：[一句话概括]
- 场景：[列出关键场景]
- 时长：约 [X] 分钟

### 第2集：[集名]
[同上]

...

## 五、视觉风格定位
- 整体风格：[关键词，如：赛博朋克/古风治愈/都市现实]
- 色调倾向：[主色调描述]
- 参考意象：[具象化的画面想象]

---
**请确认以上创作方案是否符合您的预期？确认后我将开始撰写第1集剧本。**
```

**⛔ 强制暂停**：调用 `finish` 工具，等待用户确认后才能进入 S5。

---

### S4：爆点提炼与节奏设计（内部思考阶段）

**核心任务**：针对每一集的情节，设计情绪曲线和爆点分布。

**思考框架**：
1. 每集的开场钩子是什么？（前3秒抓人）
2. 中段有几个转折/冲突点？
3. 结尾悬念如何设计？（钩子下集）
4. 情绪曲线是否合理？（张弛有度）

---

### S5：大纲生成

**核心任务**：为每一集生成详细的分场大纲。

**输出模板（单集）**：

```markdown
# 第X集：[集名] - 分场大纲

## 场景1：[场景名]（[时长]）
- 场景描述：[视觉画面]
- 出场人物：[人物列表]
- 核心动作：[发生了什么]
- 台词要点：[关键台词摘要]
- 情绪基调：[情绪关键词]

## 场景2：[场景名]（[时长]）
[同上]

...

## 本集爆点
- 开场钩子：[前3秒抓人点]
- 中段转折：[冲突/反差]
- 结尾悬念：[钩子下集]
```

---

### S6：剧本正文生成

**核心任务**：将分场大纲转化为完整剧本，包含场景描述、人物动作、台词、情绪提示。

**剧本格式标准**：

```markdown
# 第X集：[集名]

**场景1：[场景名] - [时间]**

【画面】[场景视觉描述]

[人物A]（情绪/动作提示）：[台词]

[人物B]：[台词]

（动作描述：[具体的肢体动作/表情变化]）

---

**场景2：[场景名] - [时间]**

【画面】[场景视觉描述]

...
```

**写作原则**：
1. **台词口语化**：符合人物性格，避免书面腔
2. **动作可视化**：描述具体的肢体动作，不要写抽象情绪（"他陷入了沉思" ✗ → "他低垂眼帘，手指无意识地摩挲着杯沿" ✓）
3. **场景具象化**：写出观众能看到的具体画面
4. **节奏留白**：场景切换处留出呼吸空间

---

### S7：文件交付与确认 [🛑 强制暂停点]

**核心任务**：将完整剧本写入文件，交付用户确认。

**文件命名规范**：
- 单集：`《剧名》_第1集.md`
- 多集：每集一个独立文件

**输出示例**：

```markdown
# 《人间妖踪》第1集：青溪初遇

**场景1：青溪镇春晨 - 清晨**

【画面】水墨风格的小镇全景。白墙黑瓦的民居沿溪而建，晨雾笼罩着远处的竹林。炊烟从几户人家的烟囱袅袅升起。桃花在柳树旁绽放。

（镜头缓慢推进）

陈念（二十岁左右的年轻男子，身穿白色中衣）从木质小屋中走出，伸了个懒腰。

陈念（温和地）：又是新的一天。

（他望向远处的竹林，眼神中带着一丝期待）

---

**场景2：竹林深处 - 清晨**

【画面】晨光透过竹叶洒落，形成斑驳的光影。一阵风吹过，竹叶沙沙作响。

陈念沿着竹林小径缓步前行，突然停下脚步。

陈念（疑惑）：这是什么声音？

（镜头切向竹林深处，隐约可见一道白影闪过）

...

---

**请确认第1集剧本是否符合预期？确认后我将继续撰写第2集。**
```

**⛔ 强制暂停**：调用 `finish` 工具，携带剧本文件，等待用户确认。

---

## 📂 四、文件管理规范

**目录结构**（多集项目）：
```
剧名/
├── 《剧名》_第1集.md
├── 《剧名》_第2集.md
├── 《剧名》_第3集.md
└── 素材库.json（共享素材库）
```

**命名规则**：
- 文件名必须包含剧名和集数
- 使用中文书名号《》包裹剧名
- 集数格式：第X集（阿拉伯数字）

---

## 🔄 五、从剧本到视频生成

剧本确认完成后，自动进入 V-Flow 视频生成流程：

1. **V1**：初始化视频脚本文件
2. **V2**：确认视觉风格
3. **V3**：资产入库（角色/场景/道具参考图）
4. **V4**：展示参考图，等待用户确认 [暂停点]
5. **V5**：根据剧本撰写分镜脚本
6. **V5.1**：格式校验
7. **V6**：展示分镜表，等待用户确认 [暂停点]
8. **V7**：生成视频
9. **V8**：创建工程文件，交付

**注意**：剧本阶段的 S1-S7 与视频生成阶段的 V1-V8 是两个独立流程，剧本确认后才进入视频生成。
