# 首尾帧模式指南

> **所属 Skill**: gen-video
> **核心用途**: 规范首尾帧模式（First-Last Frame）的视频生成流程
> **何时读取**: V5 阶段识别到"一镜到底"/"平滑转场"/"连续运动"需求时
> **前置理解**: 首尾帧模式与多素材模式互斥，只能二选一

---

## 🎯 一、模式识别与触发信号

**何时使用首尾帧模式？**

| 触发信号 | 示例 |
|---------|------|
| 一镜到底 | 镜头不切换，持续跟随主体移动 |
| 平滑转场 | 前后镜头之间要连贯，如从室内走向室外 |
| 连续运动 | 角色一直走/一直跑/一直飞 |
| 无缝衔接 | shot 1 的结尾 = shot 2 的开头（画面连续） |

**何时使用多素材模式（默认）？**

| 场景特征 | 示例 |
|---------|------|
| 跳切剪辑 | 不同场景/时间点之间的硬切换 |
| 对话交替 | 正反打对话，镜头来回切换 |
| 多场景叙事 | 同一 shot 不需要与下一个 shot 连贯 |

---

## 🔀 二、首尾帧模式 vs 多素材模式

| 维度 | 多素材模式（默认） | 首尾帧模式 |
|------|------------------|-----------|
| **resource_inputs role** | `reference_image` | `first_frame`（+ 可选 `last_frame`） |
| **素材引用** | prompt 中用 `[图片N]` 引用多张参考图 | **不用 `[图片N]`**，首帧图已包含完整画面 |
| **首帧图** | 不需要 | **V5-FF 阶段必须为每个 shot 生成首帧图** |
| **尾帧图** | 不需要 | 可选，用于控制 shot 结尾画面 |
| **转场衔接** | 无显式支持 | **shot N 的 last_frame = shot N+1 的 first_frame（同一张图）** |
| **运镜自由度** | 高，可任意描述运镜 | 受首尾帧约束，运镜路径由引擎插值 |
| **适用场景** | 常规叙事、对话、多场景 | 连续运动、长镜头、无缝转场 |

## 📐 三、首帧图生成规范（V5-FF 阶段）

### 3.1 何时生成首帧图？

**V5-FF 阶段触发条件**：
- 已确认使用首尾帧模式
- 多素材模式的 V4 已完成（或跳过）
- 需要为每个 shot 生成首帧图

### 3.2 首帧 Prompt 公式

**核心原则**：首帧图必须包含该 shot 的完整画面信息（角色+场景+构图+光影）。

**公式**：
```
[景别]，[视角]，[主体的静止姿态/位置]，[环境空间描述]，[光影细节]，[画幅比例]
```

**示例（Shot 1 首帧）**：
```
全景，从高处俯瞰，
陈念站在小屋门口，面向镜头，双手自然下垂，
白墙黑瓦的民居，青石板路延伸至远处，晨雾笼罩，
晨光从左侧打来，在地面投下柔和阴影，
16:9
```

**示例（Shot 2 首帧，与 Shot 1 衔接）**：
```
中景，平视，
陈念已经走到小溪边，背对镜头，望向对岸竹林，
小溪潺潺，桃花枝条探入画面，对岸竹林在晨雾中若隐若现，
晨光从右侧打来，水面反射微光，
16:9
```

### 3.3 尾帧图生成（可选）

**何时需要尾帧图？**
- shot 需要精确控制结尾画面
- shot 之间需要无缝衔接（shot N 的尾帧 = shot N+1 的首帧）

**尾帧 Prompt 公式**：
```
[景别]，[视角]，[主体在 shot 结尾时的姿态/位置]，[环境空间描述]，[光影细节]，[画幅比例]
```

**示例（Shot 1 尾帧 = Shot 2 首帧）**：
```
中景，平视，
陈念的背影，已经走到小溪边，背对镜头，望向对岸竹林，
小溪潺潺，桃花枝条探入画面，对岸竹林在晨雾中若隐若现，
晨光从右侧打来，水面反射微光，
16:9
```

## 🎬 四、转场实现方法

### 4.1 无缝衔接原理

**核心逻辑**：
```
shot 1: first_frame=图A, last_frame=图B   ← 图B是过渡帧
shot 2: first_frame=图B, last_frame=图C   ← 图B复用！
shot 3: first_frame=图C, last_frame=图D
shot 4: first_frame=图D                   ← 最后一个shot可以没有last_frame
```

**关键**：shot N 的 last_frame 必须与 shot N+1 的 first_frame **是同一张图**（URL 相同）。

### 4.2 生成流程示例

**场景**：陈念从家门口走到小溪边，需要 3 个 shot 无缝衔接。

**Step 1：规划分镜**
- Shot 1：家门口 → 沿街道走（全景 → 中景）
- Shot 2：沿街道走 → 走到小溪边（中景）
- Shot 3：小溪边停步 → 望向竹林（中景 → 特写）

**Step 2：生成关键帧**
- 生成图A（Shot 1 首帧）：家门口全景，陈念站在门口
- 生成图B（Shot 1 尾帧 = Shot 2 首帧）：街道中景，陈念在街道中段
- 生成图C（Shot 2 尾帧 = Shot 3 首帧）：小溪边中景，陈念背对镜头
- 生成图D（Shot 3 尾帧）：竹林特写，陈念的侧脸

**Step 3：写入分镜**

Shot 1:
```json
{
  "shot_id": 1,
  "resource_inputs": [
    { "media_type": 2, "value": "https://图A的URL", "key": "家门口", "role": "first_frame" },
    { "media_type": 2, "value": "https://图B的URL", "key": "街道中段", "role": "last_frame" }
  ],
  "aigc_prompt": "镜头从家门口全景缓慢推进，陈念沿街道向前走。不添加字幕，不添加BGM",
  "target_duration_ms": 8000
}
```

Shot 2:
```json
{
  "shot_id": 2,
  "resource_inputs": [
    { "media_type": 2, "value": "https://图B的URL", "key": "街道中段", "role": "first_frame" },
    { "media_type": 2, "value": "https://图C的URL", "key": "小溪边", "role": "last_frame" }
  ],
  "aigc_prompt": "陈念继续沿街道走，镜头跟随，直到走到小溪边。不添加字幕，不添加BGM",
  "target_duration_ms": 8000
}
```

Shot 3:
```json
{
  "shot_id": 3,
  "resource_inputs": [
    { "media_type": 2, "value": "https://图C的URL", "key": "小溪边", "role": "first_frame" },
    { "media_type": 2, "value": "https://图D的URL", "key": "竹林特写", "role": "last_frame" }
  ],
  "aigc_prompt": "陈念停步，转身侧对镜头，望向对岸竹林。镜头缓慢推近。不添加字幕，不添加BGM",
  "target_duration_ms": 8000
}
```

## ⛔ 五、常见错误与规避

### 错误1：首帧图包含多个场景

❌ **错误做法**：
- 首帧 Prompt：`陈念从家门口走向小溪边`
- 原因：一个 shot 只能有一个场景空间

✅ **正确做法**：
- 拆分为多个 shot，每个 shot 一个场景
- 通过首尾帧衔接实现场景切换

---

### 错误2：首尾帧 URL 不一致

❌ **错误做法**：
- Shot 1 的 last_frame: `https://url1/`
- Shot 2 的 first_frame: `https://url2/`（即使是相似画面）

✅ **正确做法**：
- 必须是**完全相同的 URL**
- 生成尾帧后，**复用同一张图的 URL** 作为下一 shot 的首帧

---

### 错误3：首帧图缺少关键元素

❌ **错误做法**：
- Shot Prompt：`陈念在小溪边望向竹林`
- 首帧图：只有小溪和竹林，没有陈念

✅ **正确做法**：
- 首帧图必须包含 Prompt 中描述的所有视觉元素
- 角色、场景、关键道具都要在首帧图中出现

---

### 错误4：Prompt 使用 `[图片N]` 引用

❌ **错误做法**：
```
aigc_prompt: "陈念[图片1]站在小溪边..."
```

✅ **正确做法**：
```
aigc_prompt: "陈念站在小溪边，望向对岸竹林..."
```
- 首尾帧模式下，**不用 `[图片N]` 引用**
- 首帧图已经包含了完整的画面信息

## 📋 六、首尾帧模式检查清单

| # | 检查项 |
|---|--------|
| 1 | 每个 shot 都有 first_frame |
| 2 | 需要衔接的 shot，last_frame URL = 下一 shot 的 first_frame URL |
| 3 | 首帧图包含该 shot 的所有视觉元素（角色+场景+关键道具） |
| 4 | 首帧图 Prompt 不使用 `[图片N]` 引用 |
| 5 | 首帧图画幅与视频画幅一致（如 16:9） |
| 6 | aigc_prompt 不包含与首帧图冲突的描述 |

---

## 🔄 七、V5-FF 流程总结

**标准流程**：
1. 识别首尾帧模式需求
2. 规划分镜，确定每个 shot 的首帧/尾帧画面
3. 为每个关键帧生成参考图（text_to_images 或 image_to_images）
4. 将关键帧入库到素材库
5. 写分镜时，resource_inputs 填入 `first_frame` 和 `last_frame`
6. 校验（V5.1）→ 确认（V6）→ 生成（V7）→ 交付（V8）

**⚠️ 重要提醒**：首尾帧模式与多素材模式互斥，确认使用首尾帧模式后，不再执行多素材模式的 V4 参考图确认流程，直接进入 V5-FF 首帧图生成。
