# 场景规则：数字人口播 (digital_human_talking)

> **所属 Skill**: gen-video
> **scene-rule ID**: digital_human_talking
> **核心策略**: 高密度信息轰炸与信息可视化
> **适用场景**: 知识科普、产品介绍、品牌宣传、口播视频

---

## 🎯 核心特征

**数字人口播的核心目标是高效传递信息**。画面设计必须服务于信息传递，通过高频视觉变化、清晰的信息层级、强节奏的剪辑来保持观众注意力。

---

## 📋 规则详情

| 维度 | 规则 |
|------|------|
| **时间设计** | 多个信息点合并在一个长镜头（10-15s）的分时序段落中，每段 3-5 秒 |
| **叙事设计** | 三段式闭环结构：开头钩子 → 中段展开 → 结尾金句升华 |
| **剪辑节奏** | 高频刺激节奏。画面变化频率控制在 2-3 秒/次 |
| **文案风格** | 禁止使用连接词和语气词；高频使用绝对词汇；采用指令型句式 |
| **音频设计** | 每一个信息点必须有声音+画面双重锚定 |
| **排版设计** | 三种基础画面结构：黑屏信息结构、人物主画面结构、内容主画面结构 |
| **BGM** | ❌ 不加 |
| **音色管线** | 🅱️ **管线 B（预制音色）** |

---

## 🎬 三段式叙事结构

### 第一段：开头钩子（前 3-5 秒）

**目标**：用强冲击力的信息抓住观众注意力。

**常用手法**：
- 痛点直击："你是不是也遇到过这个问题？"
- 数据冲击："90% 的人都不知道这个技巧"
- 反常识："其实，你一直都用错了"
- 悬念钩子："今天我要告诉你一个秘密"

**示例**：
```
Shot 1（12s）：
0-3s：【黑屏信息结构】画面全黑，中央出现大字："90%的人都做错了"。旁白说："90%的人都做错了。"
3-6s：【人物主画面】角色[图片1]居中，手势强调。旁白说："你可能就是其中之一。"
6-9s：【人物主画面】角色微微前倾，眼神真诚。旁白说："但今天，我要帮你改正。"
9-12s：【黑屏信息结构】画面渐黑，中央出现标题："三步解决你的焦虑"。
不添加字幕，不添加BGM
```

### 第二段：中段展开（主体部分）

**目标**：清晰、高效地传递核心信息。

**常用手法**：
- 分点列举：第一、第二、第三
- 步骤演示：第一步、第二步、第三步
- 对比展示：错误做法 vs 正确做法
- 数据支撑：用具体数字增强说服力

**示例**：
```
Shot 2（15s）：
0-4s：【人物主画面】角色[图片1]举起一根手指。旁白说："第一步，找到你的核心焦虑。"
4-8s：【内容主画面】左侧出现文字："核心焦虑清单"，右侧是清单图示。旁白说："把它写下来，不要逃避。"
8-12s：【人物主画面】角色做出手势，表示"切除"。旁白说："第二步，切断无效社交。"
12-15s：【内容主画面】画面切换为社交网络图标被划掉的动画。旁白说："你的时间很宝贵。"
不添加字幕，不添加BGM
```

### 第三段：结尾金句升华（最后 3-5 秒）

**目标**：用金句升华主题，留下记忆点。

**常用手法**：
- 金句总结："记住，焦虑不可怕，逃避才可怕"
- 行动号召："现在就开始行动"
- 反转升华："其实，真正的自由，从接纳自己开始"

**示例**：
```
Shot 3（8s）：
0-4s：【人物主画面】角色[图片1]眼神真诚，微笑。旁白说："记住，焦虑不可怕，逃避才可怕。"
4-8s：【黑屏信息结构】画面渐黑，中央出现大字："今天就开始行动"。旁白说："今天就开始行动。"
不添加字幕，不添加BGM
```

## 🖼️ 排版设计详解

### 模式 1：黑屏信息结构

**用途**：开头钩子、段落切换、痛点强调

**布局**：
- 背景：纯黑
- 人物：裁切为圆形居中（占 20%-30%）
- 大字报：下方（占 60%-80%）

**示例**：
```
0-3s：【黑屏信息结构】画面全黑，中央出现圆形人物特写（角色[图片1]），下方大字："90%的人都做错了"。
不添加字幕，不添加BGM
```

**Prompt 示例**：
```
纯黑背景，中央圆形区域展示角色[图片1]的面部特写，圆形下方是白色大字："90%的人都做错了"。16:9
```

### 模式 2：人物主画面结构

**用途**：常规讲解

**布局**：
- 人物：中近景居中（占 ≥70%）
- 信息表达：根据复杂度分支
  - 简单信息：人物上方悬浮文字
  - 中等信息：人物左侧或右侧出现卡片
  - 复杂信息：切换为内容主画面结构

**示例**：
```
0-4s：【人物主画面】角色[图片1]中近景居中，上方悬浮文字："第一步：找到核心焦虑"。旁白说："第一步，找到你的核心焦虑。"
不添加字幕，不添加BGM
```

**Prompt 示例**：
```
角色[图片1]中近景居中，背景简洁（纯色或渐变），上方悬浮白色文字："第一步：找到核心焦虑"。16:9
```

### 模式 3：内容主画面结构

**用途**：操作演示、产品细节、案例过程

**布局**：
- 内容：全屏展示（占 ≥80%）
- 人物：缩小至左下角（占 10%-20%）

**示例**：
```
0-4s：【内容主画面】中央全屏展示"焦虑清单"图示，左下角缩小展示角色[图片1]。旁白说："把它写下来，不要逃避。"
不添加字幕，不添加BGM
```

**Prompt 示例**：
```
中央全屏展示白色卡片，卡片上是"焦虑清单"的文字列表，背景简洁，左下角圆形区域展示角色[图片1]的面部特写。16:9
```

## 💬 文案风格要点

### 1. 禁止连接词和语气词

❌ **错误示例**：
```
"其实呢，这个方法很简单，就是..."
```

✅ **正确示例**：
```
"这个方法很简单。记住三点。"
```

### 2. 高频使用绝对词汇

**常用绝对词汇**：
- 必须、一定、绝对
- 永远、从不、总是
- 所有、全部、没有例外

**示例**：
```
"这一步，必须做到。没有例外。"
```

### 3. 采用指令型句式

**示例**：
- ❌ "你可以试着..." → ✅ "试着..."
- ❌ "我觉得你应该..." → ✅ "你要..."
- ❌ "这可能会帮助你..." → ✅ "这能帮你..."

---

## 🎵 音频设计要点

**核心原则**：每一个信息点必须有声音+画面双重锚定。

**原因**：
- 声音传递信息（旁白/台词）
- 画面强化记忆（文字/图示）
- 双重锚定提高信息留存率

**示例**：
```
0-4s：【人物主画面】角色[图片1]手势强调，上方文字："第一步"。旁白说："第一步，找到核心焦虑。"
```

**双重锚定**：
- 声音：旁白说"第一步，找到核心焦虑"
- 画面：文字显示"第一步"

## ⚠️ 管线 B 音色注入

**数字人口播必须使用管线 B（预制音色）**，因为需要预生成音频进行口型同步。

**V3 阶段入库**：
```bash
python gen-video/scripts/update_material_files.py "./素材库.json" \
  --role "讲解员" "https://s.coze.cn/image/xxx/" "正面" \
  --voice "女声，专业清亮，语速明快" \
  --tts \
  --voice-name "温柔女声 - vivi"
```

**V5.5 阶段 TTS 注入**：
```python
text_to_speech(
    sentences=[
        {
            "text": "90%的人都做错了。",
            "voice_name": "温柔女声 - vivi",
            "voice_style": "专业清亮，语速明快"
        },
        {
            "text": "你可能就是其中之一。",
            "voice_name": "温柔女声 - vivi",
            "voice_style": "专业清亮，语速明快"
        }
    ]
)
```

## ⛔ 常见错误与规避

### 错误1：文案冗长

❌ **错误示例**：
```
"其实呢，我觉得这个问题非常普遍，很多人都遇到过，所以今天我想跟大家分享一下我的看法..."
```

✅ **正确示例**：
```
"这个问题很普遍。今天分享三个解决方法。"
```

---

### 错误2：画面单调

❌ **错误做法**：
- 全程人物主画面，无切换
- 同一个画面持续 15 秒以上

✅ **正确做法**：
- 画面变化频率 2-3 秒/次
- 三种排版结构交替使用

---

### 错误3：缺少文字锚定

❌ **错误示例**：
```
0-4s：【人物主画面】角色[图片1]说："第一步，找到你的核心焦虑。"
```

✅ **正确示例**：
```
0-4s：【人物主画面】角色[图片1]手势强调，上方文字："第一步"。旁白说："第一步，找到你的核心焦虑。"
```
