# 视频生成工作流程

> **所属Skill**: gen-video
> **核心用途**: 视频生成 V1-V8 完整执行链路与状态调度
> **何时读取**: 执行视频生成任务时
> **前置依赖**: 无（video-script-overview.md 在 V5 阶段按需读取）

---

## 🔀 双模式路由决策树

根据用户是否上传视频素材，强制分流：

### 模式 1：用户上传视频 (source_type: 2)
- **链路**：`V1` → `V2` → `V3` → *(跳过V4)* → `V5` → `V5.1` → `V5.2` → `V6` → *(跳过V7)* → `V8`
- **约束**：`source_type`=2；`resource_inputs` 长度只能为1，且 `media_type`=1。
- **V3 提资产**：用户视频按下方"视频入库模板"写入素材库。

### 模式 2：AIGC 生成 (source_type: 1)（默认）

* **链路**：`V1` → `V2` → `V3` → `V4` → `V5` → `[V5-FF]` → `V5.1` → `V5.2` → `[V5.5 TTS]` → `V6` 🛑 → `V6.5 BGM` → `V7` → `V8`
* **约束**：`source_type`=1；必须生成参考图；`resource_inputs.value` 填素材库中的完整 URL。

### 模式 3：用户视频剪辑/卡点混剪

* **链路**：`V1` → `V2.5 BGM(可选)` → `V3(视频入库)` → `build_tracks.py` → `V8` → 交付
* **触发**：用户上传视频并要求剪辑/混剪/重新编排。跳过 V4/V5/V7。

### 视频入库模板（模式 1 和模式 3 共用）

```json
{"material_id":"mat_video_001","base_info":{"name":"用户上传视频","tag_category":4,"type_category":1,"source_category":1},"details":{"other_details":{"type_category":1,"video_file":{"file":{"url":"[视频URL]","name":"[文件名.mp4]"}}}}}
```
⚠️ `type_category` 在 base_info 和 other_details 中都要写（1=Video, 2=Image, 3=Audio）。`file.name` 不能为空。

### 模式 3 补充规则
- `project_tracks`：只写有内容的轨道，空轨道删除。每条轨道用 `TargetTime` 控制时间线位置
- **必须调 `create_video_project`** 生成 .cvp

## ⏭️ 状态转移清单（每步做完后查下一步）

| 当前完成 | 下一步 | 可执行动作 |
|----------|--------|-----------| 
| V1 初始化 | V2 | `python gen-video/scripts/get_style_prompt.py --list` 然后 `python gen-video/scripts/get_style_prompt.py <编号> --full` → 推断风格 → **不暂停，直接进入 V3**（风格信息在 V6 一并展示） |
| V2 风格确认 | V2.5 **或** V3 | **scene-rule 标注"BGM 前置"** → V2.5。**否则** → 直接 V3 |
| V2.5 BGM+分析 | V3 | ① 选曲 `select_music` 或生曲 `gen_music` ② `python gen-video/scripts/analyze_bgm.py --bpm <BPM> --duration-ms <时长ms>` ③ 写入 bgm_list（模板：`{"id":"","name":"<歌名>","author":"<作者>","duration_ms":<毫秒>,"volc_vid":"<VolcVID或空>","url":"<gen_music的url或空>"}`）④ 详见 `gen-video/references/bgm-guide.md` |
| V3 资产入库 | V4 | ① `python gen-video/scripts/update_material_files.py --list` ② 读 `gen-video/references/asset-extraction-guide.md` ③ **⛔ 读 `gen-video/references/prompt-writing-guide.md`**（生图 prompt 规范，含禁止拟人化等关键规则） ④ 入库：`--role "名称" "URL" "备注" --voice "..." [--tts]` / `--scene "名称" "URL" "备注"` ⑤ 生成参考图 |
| V4 参考图完成 | **V4 🛑** | **finish 携带所有参考图**，展示风格+角色+场景，等用户确认。用户满意 → V4.5；不满意 → 回 V3 重新生图 |
| V4 用户确认 | V4.5 | `python gen-video/scripts/gen_material_summary.py "素材库.json"` |
| V4.5 素材总览 | V5 | 依据优先级读取：**[核心必读]** ① 对应 scene-rule ② `shot-script-guide-core.md` ③ scene-rule 中指定的 golden example JSON（⛔ 必须读，写分镜时严格对齐其结构）。**[按需读取]** ④ 若为首尾帧模式，额外读 `firstframe-guide.md`。读完后分批写分镜 |
| V5 每批写完 | V5-FF **或** V5.1 | **首尾帧模式** → V5-FF 生成首帧图。**多素材模式** → 直接 V5.1 |
| V5-FF 首帧生成 | V5.1 | 仅首尾帧模式。为每个 shot 调 `image_to_images` 或 `text_to_images` 生首帧图 → 写入 resource_inputs (role=first_frame)。详见 `firstframe-guide.md` |
| V5.1 校验 | V5.2 | `python gen-video/scripts/validate_video_script.py "xxx.cvs" --auto-fix`。通过后：`python gen-video/scripts/validate_fidelity.py "剧本.md" "xxx.cvs"`（可选） |
| V5.2 通过 | V5.5 **或** V6 | **多素材模式 + 管线 B（预制音色）** → V5.5 TTS 注入。**首尾帧模式或管线 A** → 直接 V6（⛔ 首尾帧只能走管线 A） |
| V5.5 TTS 注入 | **V6 🛑** | 仅多素材模式 + 管线 B。**⛔ 必读 `references/voice-library.md` 的 B3-B6 步骤，严格照搬代码模板**——不要自己写注入逻辑。顺序：B3 生成音频 → B4 入库素材库 → B5 注入 resource_inputs（role 必须是 `reference_audio`，media_type 必须是 3）→ B6 更新 prompt 引用 → 重跑 V5.1 校验。**⛔ TTS 音频时长必须 ≥1.8s** |
| V5.5 或 V5.2 | **V6 🛑** | **finish 同时发：① 所有参考图文件 ② .cvs 分镜表 ③ .md 剧本（如有）**。用户一次性审核素材+分镜，等待指示 |
| V6 用户确认 | V6.5 **或** V7 | **BGM 前置（已在 V2.5 完成）** → 直接 V7。**否则** → V6.5。**⛔ 总时长 < 15s 时跳过 BGM** |
| V6.5 BGM（后置） | V7 | 选曲/生曲 → 写入 bgm_list。**总时长 < 15s 时跳过** |
| V7 视频生成 | V8 **或** V7 自环 | 调平台工具 `batch_gen_video_shots`。**[前置条件]**：output=null 的 shot；bgm_list 已处理；auto-fix 已跑完。**⛔ 重试前必须 `cp xxx.cvs xxx_backup.cvs` 备份**——batch_gen 失败时可能覆盖已成功 shot 的 output，失败后用 Python 从 backup 恢复已成功 shot 的 output 再重试。**[判定]**：① 全部成功 → V8。② 有失败（含排队失败）→ **先看错误类型**：排队失败/超时/服务繁忙 → 直接置 null 重试即可，不需要改 prompt 或素材；`SensitiveContentDetected` → 真人检测问题，按 asset-extraction-guide 的真人检测方案处理。⛔ 最多重试 2 次（含排队失败的重试）。③ **重试 2 次仍有失败 → 无论成功率多少，立即 V8 create_video_project 把已成功的 shot 做成工程交付**，同时告知用户哪些 shot 失败。用户拿到的是"有缺口的成品"而不是"什么都没有"。已成功的视频是用户资产，不能因为少数失败就全部重来。**⛔ 绝对禁止空手报错——必须先交付工程文件，再说明失败情况。** **V7 后不存在语音步骤。** |
| V8 建工程 | V8.5 **或** 交付 | 调 `create_video_project`。前提：V7 已完成。**⛔ create_video_project 会重建 project_tracks，之前对 tracks 的手动修改会被覆盖**。需要修饰 → V8.5；否则 finish 交付。（**⛔ 绝对禁止发素材库.json**） |
| V8.5 轨道编辑 | 交付 | **⛔ 必须用 `edit_tracks.py` 编辑轨道，禁止手写 Python 操作 project_tracks 索引**。BGM 超出视频时长 → `edit_tracks.py --trim-bgm`。裁剪/变速/删段 → `edit_tracks.py --trim/--speed/--remove-shot`。操作完 finish .cvs |

## 🔧 异常诊断（仅 P0，P1/P2 见 shot-script-guide-errors.md）

| 故障 | 修复 |
| --- | --- |
| **tracks 保护** | `auto-fix` 只在 `project_tracks=null` 时重置为 `[]`，**非空 tracks 不受影响**。E-Flow ①②级编辑不需要跑 validate（见 SKILL.md 四级编辑表），③④级可以正常跑 |
| **TTS 分管线** | 管线 A 禁止调 `text_to_speech`（batch_gen 自动合成）。管线 B 调 TTS 后音频**必须先入素材库**再注入 resource_inputs |
| **修复死循环** | 同一 shot 修 ≥3 轮 → finish 暂停向用户说明 |
| **create_video_project BGM 上传失败** | 报 `unsupported protocol scheme` 或 `uploadAudio xxx.mp3 failed` → BGM 引用了本地文件名而不是 URL。**必须同时清理两处**：① `bgm_list = []` ② `project_tracks` 中 Type=audio 的轨道整条删除。只清 bgm_list 不清 tracks 会反复失败 |
| **排队失败** | batch_gen 返回"排队失败"→ 服务繁忙，自动重试（最多 2 次）。⛔ 不要改 prompt 或素材，直接置 null 重试。超过 2 次 → 交付已有成果 |
