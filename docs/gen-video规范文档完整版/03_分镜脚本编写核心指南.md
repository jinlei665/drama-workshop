# 分镜脚本编写执行手册 (V5 核心引擎)

> **前置依赖**: 必须已执行 V4.5 生成素材总览。
> **参考示例**: 多素材模式的完整 shot 示例见 `gen-video/references/shot-script-guide-errors.md`
> **核心任务**: 将剧本转化为带有时序、运镜和精准资产引用的合法 `.cvs` (JSON) 文件。

## ⛔ 写每个 shot 前默念三遍

1. **prompt 里出现的每个角色/场景名后面必须跟 `[图片N]`**，同场景在场的角色一个都不能漏
2. **先写台词 → 按字数算时长 → 累加得 shot 总时长**，禁止先定时长再塞台词
3. **一个 shot 只放一张场景图**，角色图可以多张

---

**⚠️ BGM 前置追溯**：如果当前环境存在 `beat_map.json`（BGM 前置场景），后续的时长切分必须受其驱动（见下方音乐驱动模式）。

## 🔢 一、数学与时间铁律 (计算时长与拆分)

**0. ⛔ 场景审计（写分镜前必做）**：

shot 数量由**场景/空间数量**决定，不由情节点数量决定。写分镜前先做：

```
第一步：列出所有不同的物理空间/场景（如"驾驶舱""虫洞深空""对接舱"）
第二步：每个空间 = 1 个 shot（同空间的多个情节点用分时序实现）
第三步：合并检查——相邻场景如果服务于同一情绪节拍且各自 <8s，合并为 1 个 shot 的多段分时序
第四步：转折标记——标出核心转折/高潮所在的 shot，景别优先分配近景/特写
第五步：跳转标记——相邻场景有重大时空跳转（重生/穿越/闪回/时间跳跃）时，在前一个 shot 末尾加视觉过渡（画面渐白/碎裂/色调骤变），不能硬切
第六步：shot 数 ≈ 合并后的场景数
```

| ❌ 错误思维 | ✅ 正确思维 |
|-----------|-----------| 
| 8 个故事节拍 → 8 个 shot | 4 个不同场景 → 4 个 shot，每个 shot 内 2-3 个分时序段落 |
| 每个 shot 7-8 秒，节奏紧凑 | 每个 shot 12-15 秒，内部用分时序控制节奏快慢 |
| 石桥(9s)+庭院(6s) 各一个 shot | 石桥+庭院都是"甜蜜"→合并为 1 shot 15s，两段分时序 |
| 核心转折全用中景/全景 | 核心转折 shot 必须包含至少 1 个近景/特写段落（放大情绪冲击） |

**1. 目标对齐**：先评估剧本总时长。1分钟视频约 4-6 个 shot，3分钟约 12-18 个。**⛔ 这是参考范围，实际 shot 数以场景审计结果为准。如果审计出的场景数明显超出参考范围（如 60s 有 7+ 场景），必须执行第三步合并。**

**2. 单 Shot 时长计算公式 (必须为整千数！)**：
- **纯空镜 / 简单动作**：固定 6000ms - 8000ms。
- **复杂动作链**：固定 10000ms - 12000ms。
- **有台词/旁白 (严密计算)**：台词字数 × 300ms + 500ms缓冲。**算完后必须向上取整到整千**（例：计算得 7400ms ➡️ 强制写为 8000ms）。绝对上限为 15000ms。
  - **⛔ 分时序段落写作顺序**：先写完每段的画面+台词内容 → 含台词的段落按「字数×0.3+0.5」定时长、纯画面段落按节奏3-5s → 累加所有段落得 shot 总时长 → 写 target_duration_ms。**禁止反过来（先定总时长再平均切段再塞台词）——这会导致每段语速不一致。**

**3. 强制拆分红线 (Shot_id 递增条件)**：
- 物理空间/场景切换。
- 时间跳跃。
- **单句台词超过 40 字**（计算得 > 12500ms，逼近上限）→ 按语义断句拆为两个 Shot：
  - 拆分点选在逗号/句号处，确保语义完整
  - 两个 Shot 复用同一张场景图和角色图（resource_inputs 相同）
  - 前一个 Shot 以"角色说完前半句，表情凝重"类过渡描述结尾
  - 后一个 Shot 以"角色继续说"开头，保持画面连续性
- **⛔ 空镜/旁白开场上限**：纯空镜+旁白的开场段总时长 ≤ 总片长 15%。如 60s 片子开场空镜最多 10s。旁白太长时**精简旁白**，而不是拆成多个空镜 shot。
*(注意：正反打对话、连续动作链、同场景新角色入画，保持同一 Shot_id，用分时序切分！)*

**⚠️ 音乐驱动模式（当存在 beat_map.json 时，以上内容驱动规则被覆盖）**：

scene-rule 标注"BGM 前置"时，V2.5 会生成 `beat_map.json`。此时 shot 时长和切分改用以下规则：
1. **shot 边界**：从 beat_map 的 `recommended_shot_cuts` 中选取，相邻 cut 间隔 < 4s 则合并
2. **target_duration_ms**：两个相邻 cut 的间隔，向上取整到整千，范围 [4000, 15000]
3. **分时序段落**：每个 beat 对应一个段落，段落时长 ≈ `beat_interval_ms`，向上取整到整秒
4. **section→视觉强度**：beat_map.sections 的 energy 字段指导视觉强度——low→慢镜/空镜，medium→正常叙事，high→快切/特效

## 🎯 二、素材引用与数组对齐铁律 (防崩核心)

**⛔ 致命错误警告：大模型极易在此处将引用的 [图片N] 与 JSON 数组顺序搞混！写分镜前必须强制脑内预演以下映射关系！**

| Prompt 中的引用标识 | 在 JSON `resource_inputs` 数组中的绝对位置 |
|-------------------|------------------------------------------|
| `[图片1]` | 必须是数组的**第 1 个**元素 (Index 0) |
| `[图片2]` | 必须是数组的**第 2 个**元素 (Index 1) |
| `[图片3]` | 必须是数组的**第 3 个**元素 (Index 2) |
*(绝对禁止出现 `[图片0]`！数组元素排列顺序必须与 Prompt 中 `[图片N]` 首次出现的顺序完全一致！)*

**引用规范**：
1. **见名必引**：Prompt 中描述的任何视觉主体（角色/具体场景/道具），必须挂载来自 V4.5 素材总览的 `[图片N]`。
2. **在场必写**：剧本中同一场景内有台词或明确动作的角色，**全部必须出现在该 shot 的 prompt 和 resource_inputs 中**。不能因为某段分时序只聚焦角色 A 就省略同场景在场的角色 B——否则观众会问"另一个人去哪了"。如果 anime_short 规则限制单画面 ≤3 角色，用分时序分段切换焦点角色，而不是直接砍掉。
3. **无图则糊**：如果素材总览里没有该物体的参考图，**绝对禁止**在 Prompt 里精细描述它的外观，只写通用名词（如：一辆摩托车）。
4. **单场景铁律**：每个 shot 的 resource_inputs 中**最多只引用 1 张场景图**。引用两张不同场景图会导致画面融合冲突。如果画面需要"从 A 地看向 B 地"（如从教室窗户望操场），**只引用 A 地场景图**（角色所在的空间），B 地靠 prompt 文字描述或者用一张包含两者的合成场景图。
5. **景别匹配**：Prompt 的景别必须与 `resource_inputs` 中的场景图景别对应。素材总览中同一场景可能有多张不同景别的参考图（全景/中景/反向/近景），选最匹配当前 shot 主景别的那张。如 Prompt 写【全景】引用全景图，写【中景】引用中景图，有正反打对话引用反向视图。
6. **标题卡/纯背景镜头**：`batch_gen_video_shots` 要求每个 shot 的 `resource_inputs` 不能为空数组。即使是黑屏标题卡、纯色片尾，也必须有至少 1 个 resource_input。做法：先用 `text_to_images` 生成一张纯色/渐变背景图，入素材库后作为 `resource_inputs[0]` 引用。

## 🎬 三、Prompt 镜头语言公式 (分时序强制)

**每个 Shot 无论长短，都必须使用分时序切分内部节奏！** (每段 3-5 秒，节奏越快分段越密)

**撰写公式**：
`[时间区间]：【景别】 + [运镜指令] + [主体[图片N]] + [可视化的动作/微表情] + [环境光影]`

运镜指令常用词：`镜头缓慢推进` / `横移` / `切镜到[角色]` / `镜头切回[角色]`。其中"切镜到/镜头切回"适合对话交替场景，比重新描述完整场景更简洁。

**规则底线**：
1. **画面必须自己讲故事**：每个分时序段落必须有**可见的物理变化**（人的动作、物体的运动、光影的变化）。如果某个段落去掉旁白后读起来像"一个人站着/坐着/看着"，必须补充具体动作（手指动作、表情变化、身体姿态转换）。**AI 视频不是配图朗读，画面要动起来。**
2. **禁止瞬移/消失**：上一秒在画面的主体，下一秒不能凭空消失，必须交代去向或保持在场。
3. **禁止抽象动词**：严禁写"激烈战斗"、"试探地说"，必须转化为**可视物理动作**（"向左闪避侧踢"、"眼神游移轻声说"）。
4. **复杂交互规避**：严禁写精确肢体接触（握手/接吻/剑抵咽喉），AI无法生成，必须改为暗示性切镜。
5. **关键信息用视觉传达**：反转、高潮、核心情节点必须通过**画面中的物理事件**让观众感知（物体状态变化、角色肢体反应），不能仅靠旁白解释。如果一个关键信息去掉旁白后无法从画面理解，说明视觉表达不够。
6. **台词挂载**：
   - **角色在画面中说话**：`角色名说："台词"`（角色名必须与素材库 `base_info.name` 完全一致，从 V4.5 素材总览获取）
   - **旁白/画外音（角色不在画面中）**：`旁白说："台词"`（前提：V3 阶段已创建旁白虚拟角色入库，name 为 `旁白`）
   - V5.1 auto-fix 会自动将 `角色名说：` 展开为完整音色公式（`角色名是XXX，说：`），**两种模式（多素材/首尾帧）统一由 auto-fix 管理，禁止手写内联音色公式**
   - **⛔ 对话格式必须是 `角色名说："台词"` 或 `角色名说道："台词"`**。禁止在角色名和"说"之间加修饰词（auto-fix 正则无法匹配会导致音色注入失败）：
     - ❌ `林渊声音低沉，说："台词"` / `陈默平静开口，说："台词"`
     - ✅ `林渊说："台词"` / `陈默说："台词"`（auto-fix 自动展开音色）
7. **Prompt 结尾防伪**：每个 `aigc_prompt` 字符串的**绝对末尾**，必须手动加上：`不添加字幕，不添加BGM`。

*(注：如果有剧本，台词内容必须 100% 逐字保留，严禁摘要或精简！但台词格式必须标准化为 `角色名说："台词"` —— 剧本里的 `角色名（修饰词）：台词` 要去掉括号修饰词并补"说"字。修饰词（如"激动地""冷笑一声"）转化为画面动作描述。非主角若多次露脸，必须分配图片引用，纯背景板可略。)*

## 💻 四、文件写入策略与 JSON 转义红线 (生死攸关)

**⛔ 前置检查：.cvs 骨架必须由 init_video_script 生成**
写分镜前先 `read_file` 读 .cvs 确认顶层有 `title`/`ratio`/`shot_scripts` 字段。如果文件不存在或顶层结构不对，**必须先调 `init_video_script` 初始化**，禁止手写 JSON 骨架（手写极易出错）。

**⚠️ 分批写入策略与工具调用铁律**：
- **单次限制**：每次生成/写入**最多 5 个 Shot**（⛔ 从 8 降到 5，减少长 JSON 生成时的后段质量衰减）。
- **所有批次统一写法**：用 `bash_tool` 执行 Python —— 读取 .cvs → 设置/追加 `data["shot_scripts"]` → `json.dump` 写回。**⛔ 禁止用 `edit_file` overwrite 整个文件**（会覆盖 bgm_list、project_tracks 等已有字段，BGM 前置场景中 gen_music 已将数据写入 .cvs）。第 1 批用 `data['shot_scripts'] = new_shots_list`，后续批次用 `data['shot_scripts'].extend(new_shots_list)`（**严禁 `.append()`**，会导致嵌套列表）。字段名是 `shot_scripts`，不是 `shots`。`shot_id` 是 Integer 不是字符串。
- **步步为营**：每批写完（无论是 write 还是追加），**必须立即执行一次 V5.1 校验**，通过后再继续下一批！
- **修复 = 改报错字段，不重写 shot**：validate 报错后用 Python 只改报错的字段（如 `shots[0]['aigc_prompt']` 的某段时长），**禁止重新构造整个 shot 的 JSON**。全量重写会丢失未报错字段的数据（resource_inputs 中的角色、已有的 key/role 映射）。
- **质量自检**：每批校验通过后，用 Python 读取刚写入的 shot，打印每个 shot 的 `shot_id / target_duration_ms / resource_inputs 数量 / aigc_prompt 前30字`，快速检查有无明显异常（如时长全相同、引用数量为 0、prompt 开头重复）。发现异常立即修复再继续。

**⚠️ 换行符**：`aigc_prompt` 中分时序之间的换行必须写 `\n`，禁止真实回车。
**⚠️ 台词引号**：台词中的中文引号 `""` 在 Python 字符串里**不需要转义**——直接写 `角色说："台词"` 即可。如果用了 `\"`（反斜杠转义），validate 的音色注入和字数检查都会失效。

### 标准 Shot JSON 结构拓扑 (第1批写入时严格遵循)
```json
{
  "shot_id": 1,
  "semantic_desc": "一句话概括场景",
  "show_type": 1,
  "source_type": 1,
  "resource_inputs": [
    { "media_type": 2, "value": "https://素材总览里的场景URL", "key": "场景名", "role": "reference_image" },
    { "media_type": 2, "value": "https://素材总览里的角色URL", "key": "角色名", "role": "reference_image" }
  ],
  "aigc_prompt": "0-4s：【全景】场景[图片1]内，角色[图片2]推门而入。\n4-8s：【中景】角色[图片2]坐下。角色说：\"你好啊。\"\n不添加字幕，不添加BGM",
  "target_duration_ms": 8000,
  "video_model": "seedance2.0_pro"
}
```
(注：narration_id 等字段将由 V5.1 auto-fix 自动补全，你在此处必须保证基础 JSON 结构的绝对闭环！)

## 📋 五、写完每个 shot 对照检查

| # | 检查项 |
|---|--------|
| 1 | 以【景别】开头，以"不添加字幕，不添加BGM"结尾 |
| 2 | ≥7s 必须有分时序（`0-4s：... 4-8s：...`） |
| 3 | `[图片N]` 从1开始，数量 = resource_inputs 长度 |
| 4 | 对话格式：`角色名说："台词"`，禁止加修饰词 |
| 5 | 台词字数不超时长上限（5s≤13字, 8s≤23字, 10s≤30字） |
| 6 | 暴力词规避：打/踢/捂/掐 → 改为暗示性动作（"做了个嘘的手势"） |
| 7 | 全片景别有变化，核心转折 shot 必须有近景/特写段落 |
