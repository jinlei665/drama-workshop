# 场景规则：知识解说 (knowledge_explanation)

> **所属 Skill**: gen-video
> **scene-rule ID**: knowledge_explanation
> **核心策略**: 广播级标准叙事
> **适用场景**: 科普视频、教程讲解、知识分享、教育内容

---

## 🎯 核心特征

**知识解说的核心目标是清晰、准确地传递信息**。画面设计必须服务于知识传递，通过标准化的叙事节奏、紧密的音画同步、可视化的知识点呈现来提高观众的理解效率和记忆留存。

---

## 📋 规则详情

| 维度 | 规则 |
|------|------|
| **节奏设计** | 三段式节奏：开场戳痛点 → 主体由浅入深 → 收尾价值输出 |
| **视觉叙事** | 画面内容需要与旁白紧密相连，特写画面不得超过 3s |
| **文案风格** | 口语化、通俗化。多用"你"、"我们"建立对话感 |
| **音频设计** | 先写旁白设计，配合整体节奏的情绪曲线 |
| **运镜** | 平稳流畅，常见固定镜头或简单横向移动 |
| **数字人处理** | 通常只在首尾片段使用数字人出镜，中间用画面直接说明 |
| **知识点可视化** | 讲解要点段落必须有**专门的知识点背景图**作为 resource_inputs |
| **旁白字数控制** | 单 shot 旁白字数必须严格对照速查表上限 |
| **BGM** | ❌ 不加 |
| **音色管线** | 🅱️ **管线 B（预制音色）** |

---

## 🎬 三段式节奏设计

### 第一段：开场戳痛点（前 10-15 秒）

**目标**：用痛点问题抓住观众注意力，建立关联感。

**常用手法**：
- 提问式："你是否也遇到过这个问题？"
- 数据式："据统计，80% 的人都不知道..."
- 场景式："想象一下，当你..."

**示例**：
```
Shot 1（12s）：
0-4s：【中景】角色[图片1]（数字人）面对镜头，眼神真诚。旁白说："你是否也遇到过这样的困扰？"
4-8s：【特写】知识点背景图[图片2]，展示"困扰清单"。旁白说："每天忙忙碌碌，却总觉得效率低下。"
8-12s：【中景】角色[图片1]微微点头，表情理解。旁白说："其实，这并不是你的错。"
不添加字幕，不添加BGM
```

### 第二段：主体由浅入深（主体部分）

**目标**：清晰、分层地传递核心知识。

**常用手法**：
- 递进式：第一步 → 第二步 → 第三步
- 对比式：错误做法 vs 正确做法
- 案例式：理论 + 实例

**示例**：
```
Shot 2（15s）：
0-4s：【全景】知识点背景图[图片3]，展示"时间管理矩阵"。旁白说："第一步，区分重要和紧急。"
4-8s：【特写】镜头聚焦矩阵的"重要但不紧急"区域。旁白说："这是大多数人忽视的区域。"
8-12s：【全景】知识点背景图[图片4]，展示"四象限法则详解"。旁白说："把80%的精力放在这里。"
12-15s：【中景】角色[图片1]手势强调。旁白说："你会发现，效率提升了不止一倍。"
不添加字幕，不添加BGM
```

### 第三段：收尾价值输出（最后 5-10 秒）

**目标**：升华主题，提供行动指引。

**常用手法**：
- 总结式："记住这三点，你就能..."
- 行动式："现在就开始行动吧"
- 升华式："真正的改变，从认知开始"

**示例**：
```
Shot 3（8s）：
0-4s：【中景】角色[图片1]微笑，眼神坚定。旁白说："记住，时间管理不是压缩时间，而是优化选择。"
4-8s：【特写】知识点背景图[图片5]，展示核心金句。旁白说："从今天开始，做时间的主人。"
不添加字幕，不添加BGM
```

## 🖼️ 知识点可视化

**核心规则**：讲解要点段落必须有**专门的知识点背景图**作为 resource_inputs。

**原因**：
- 纯人物口播难以承载复杂信息
- 知识点图示提高理解效率和记忆留存
- 画面变化保持观众注意力

**知识点背景图类型**：

| 类型 | 用途 | 示例 |
|------|------|------|
| 清单图 | 列举要点 | "困扰清单"、"时间管理矩阵" |
| 流程图 | 展示步骤 | "四步法"、"决策流程" |
| 对比图 | 对比展示 | "错误做法 vs 正确做法" |
| 数据图 | 数据支撑 | "80%的时间用于..." |
| 概念图 | 抽象概念具象化 | "心智模型"、"认知框架" |

**知识点背景图生成示例**：

```
# 时间管理矩阵
纯白背景，中央展示"时间管理矩阵"四象限图，
左上：重要且紧急（红色标记），
右上：重要但不紧急（绿色标记，重点强调），
左下：不重要但紧急（黄色标记），
右下：不重要且不紧急（灰色标记），
每个象限内有简洁文字说明，
整体设计简洁专业，无人物，16:9

# 四象限法则详解
纯白背景，中央展示"四象限法则详解"，
上方标题："把80%精力放在重要但不紧急"，
下方四个彩色方块，每个方块包含：
- 图标（放大镜/目标/闹钟/休息）
- 简短文字（"规划"/"目标"/"应付"/"浪费"）
整体设计简洁专业，无人物，16:9
```

## 💬 文案风格要点

### 1. 口语化、通俗化

**核心原则**：用日常语言，避免专业术语堆砌。

❌ **错误示例**：
```
"时间管理的核心在于对时间资源进行优化配置，以实现效率最大化..."
```

✅ **正确示例**：
```
"时间管理，其实就是把时间花在刀刃上。"
```

### 2. 建立"你"、"我们"的对话感

**核心原则**：拉近距离，增强代入感。

**示例**：
```
"你是否也遇到过..."
"我们来看看..."
"你会发现..."
```

### 3. 分层递进，由浅入深

**核心原则**：先说"是什么"，再说"为什么"，最后说"怎么做"。

**示例**：
```
"这是时间管理矩阵。"（是什么）
"为什么它有效？因为它帮你区分优先级。"（为什么）
"怎么用？记住这三步..."（怎么做）
```

## ⏱️ 旁白字数控制

**核心规则**：单 shot 旁白字数必须严格对照速查表上限。

**速查表**：

| Shot 时长 | 旁白字数上限 | 计算公式 |
|----------|------------|---------|
| 8s | ≤ 23 字 | 8s × 300ms/字 - 500ms缓冲 ≈ 23 字 |
| 10s | ≤ 30 字 | 10s × 300ms/字 - 500ms缓冲 ≈ 30 字 |
| 12s | ≤ 36 字 | 12s × 300ms/字 - 500ms缓冲 ≈ 36 字 |
| 15s | ≤ 45 字 | 15s × 300ms/字 - 500ms缓冲 ≈ 45 字 |

**示例（12s shot）**：
```
旁白说："第一步，区分重要和紧急。这是大多数人忽视的区域。把80%的精力放在这里。"
```
字数：7 + 10 + 10 = 27 字，符合 ≤ 36 字要求。

## 🎥 数字人处理

**核心规则**：通常只在首尾片段使用数字人出镜，中间用画面直接说明。

**原因**：
- 数字人口播成本较高（需预生成音频）
- 中间段落用知识点背景图更直观
- 首尾数字人出镜建立亲切感

**典型结构**：
- **开场**：数字人口播 + 痛点引入
- **主体**：知识点背景图 + 旁白
- **收尾**：数字人口播 + 价值升华

**示例**：
```
Shot 1（开场，数字人出镜）：
0-4s：【中景】角色[图片1]面对镜头。旁白说："你是否也遇到过这样的困扰？"

Shot 2（主体，知识点背景图）：
0-4s：【全景】知识点背景图[图片2]。旁白说："第一步，区分重要和紧急。"

Shot 3（收尾，数字人出镜）：
0-4s：【中景】角色[图片1]微笑。旁白说："从今天开始，做时间的主人。"
```

## 🎵 音频设计要点

**核心原则**：先写旁白设计，配合整体节奏的情绪曲线。

**情绪曲线示例**：
- **开场**：理解、共鸣（平缓）
- **主体**：启发、兴奋（递进）
- **收尾**：坚定、激励（升华）

**旁白风格建议**：
- 语速：中等偏快（科普内容）
- 情绪：专业、真诚、有感染力
- 音色：清亮、稳重、有亲和力

**管线 B 预制音色推荐**：
| 音色名 | 风格 | 适用场景 |
|-------|------|---------|
| 解说小明 | 男声，清晰 | 科普、教程 |
| 温柔女声 - vivi | 女声，温柔 | 治愈、情感 |
| 大壹 | 男声，稳重 | 专业、商务 |

## ⛔ 常见错误与规避

### 错误1：旁白字数超标

❌ **错误示例**：
- 12s shot，旁白 50+ 字

✅ **正确示例**：
- 12s shot，旁白 ≤ 36 字

**解决方案**：精简旁白，拆分长句为多个 shot。

---

### 错误2：缺少知识点背景图

❌ **错误做法**：
- 全程数字人口播，无知识点可视化

✅ **正确做法**：
- 首尾数字人出镜
- 中间用知识点背景图 + 旁白

---

### 错误3：运镜复杂

❌ **错误做法**：
```
0-5s：【全景→特写→中景】镜头快速变化
```

✅ **正确做法**：
```
0-4s：【全景】镜头固定
4-8s：【特写】镜头固定
```
