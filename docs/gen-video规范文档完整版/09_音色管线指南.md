# 音色管线指南

> **所属 Skill**: gen-video
> **核心用途**: 规范语音合成的两条管线（A 描述音色 vs B 预制音色）的选择与执行
> **何时读取**: V3 阶段资产入库时，或 V5.5 阶段 TTS 注入时
> **前置理解**: 不同场景类型对应不同的音色管线，必须在 V3 阶段确定

---

## 🎯 一、两条音色管线概览

| 管线 | 名称 | 适用场景 | 核心区别 |
|------|------|---------|---------|
| **B（默认）** | VoiceSelection（预制音色） | 预告片、氛围片、科普解说、电商带货、数字人口播、品牌宣传 | Agent 从音色库选 voice_name → TTS 生成音频 → 音频作为 reference_audio 注入分镜 |
| **A** | VoiceDescription（描述音色） | **漫剧**、**氛围片**、**预告片**、**首尾帧模式** | Agent 写自然语言描述 → batch_gen 内部合成 |

**核心差异**：
- **管线 A**：音色描述写入素材库，V5.1 auto-fix 自动展开，V7 生成时内部合成
- **管线 B**：V5.5 先 TTS 生成音频文件，音频注入分镜的 resource_inputs

---

## 📋 二、场景→管线决策表

| 场景类型 | scene-rule | 推荐管线 | 原因 |
|---------|-----------|---------|------|
| 科普/教程/解说 | knowledge_explanation | **B（预制音色）** | 专业解说需要稳定统一的音色 |
| 电商/带货 | ecommerce_marketing | **B（预制音色）** | 品牌调性统一，需要可控音色 |
| 数字人口播 | digital_human_talking | **B（预制音色）** | 口型同步需要预生成音频 |
| 漫剧/二次元 | anime_short | **A（描述音色）** | 角色音色多样，适合描述性生成 |
| 预告片 | trailer_preview | **A（描述音色）** | 台词少，情绪表达优先 |
| 氛围短片 | atmosphere_short | **A（描述音色）** | 情绪氛围优先，音色描述更灵活 |
| 首尾帧模式 | — | **A（描述音色）** | 首尾帧模式不支持音频注入 |

---

## 🅰️ 三、管线 A：描述音色流程

### 适用场景

- 漫剧/二次元短剧
- 氛围短片（情绪表达优先）
- 预告片（台词少，情绪冲击强）
- 首尾帧模式（不支持音频注入）

### V3 阶段：入库

**命令格式**：
```bash
python gen-video/scripts/update_material_files.py "./素材库.json" \
  --role "角色名" "https://图片URL" "正面" \
  --voice "自然语言描述"
```

**音色描述公式**：
```
[性别]声，[年龄段]音色，音调[特征]，音色质感[质感]，发音方式[特征]，带有[情感气质]
```

**示例**：
```bash
# 漫剧男主
python gen-video/scripts/update_material_files.py "./素材库.json" \
  --role "陈念" "https://s.coze.cn/image/xxx/" "正面" \
  --voice "男声，青年音色，温和清亮，语速中等，带有书卷气"

# 漫剧反派
python gen-video/scripts/update_material_files.py "./素材库.json" \
  --role "墨渊" "https://s.coze.cn/image/xxx/" "正面" \
  --voice "男声，中年音色，低沉沙哑，语速缓慢，带有阴冷威压"

# 旁白
python gen-video/scripts/update_material_files.py "./素材库.json" \
  --narrator "https://s.coze.cn/image/kgWvY_NvVcQ/" \
  --voice "男声，老年音色，沉稳有力，带有故事感"
```

### V5 阶段：写分镜

**台词格式**：
```
角色名说："台词"
```

**示例**：
```
陈念说："这玉佩，似乎与我有缘。"
```

**⚠️ 禁止格式**：
- ❌ `陈念声音低沉，说："台词"`（auto-fix 无法匹配）
- ❌ `陈念："台词"`（缺少"说"字，auto-fix 无法匹配）
- ✅ `陈念说："台词"`（标准格式，auto-fix 自动展开音色）

### V5.1 阶段：auto-fix 自动展开

**auto-fix 会自动将**：
```
陈念说："这玉佩，似乎与我有缘。"
```

**展开为**：
```
陈念是男声，青年音色，温和清亮，语速中等，带有书卷气，说："这玉佩，似乎与我有缘。"
```

**无需手动操作**，validate 的 `--auto-fix` 参数会自动处理。

### V7 阶段：batch_gen 内部合成

**batch_gen_video_shots** 工具会根据 prompt 中的音色描述自动合成语音，无需额外操作。

## 🅱️ 四、管线 B：预制音色流程

### 适用场景

- 科普/教程/解说
- 电商/带货
- 数字人口播
- 品牌宣传（需要统一品牌音色）

### V3 阶段：入库

**命令格式**：
```bash
python gen-video/scripts/update_material_files.py "./素材库.json" \
  --role "角色名" "https://图片URL" "正面" \
  --voice "音色描述" \
  --tts \
  --voice-name "预制音色名"
```

**参数说明**：
- `--tts`：标记使用管线 B（预制音色）
- `--voice-name`：从音色库选择的预制音色名

**预制音色库**（部分示例）：
| 音色名 | 风格 | 适用场景 |
|-------|------|---------|
| 林潇 | 男声，青年 | 通用解说、品牌宣传 |
| 咪仔 | 女声，活泼 | 电商带货、轻松内容 |
| 大壹 | 男声，稳重 | 科普教程、商务内容 |
| 温柔女声 - vivi | 女声，温柔 | 治愈内容、情感叙事 |
| 儒雅逸辰 | 男声，儒雅 | 文化内容、历史讲解 |
| 解说小明 | 男声，清晰 | 知识解说、教程 |
| 磁性解说 | 男声，磁性 | 预告片、广告 |

**完整音色列表见**：`gen-video/references/voice-library.md`

**示例**：
```bash
# 科普解说主持人
python gen-video/scripts/update_material_files.py "./素材库.json" \
  --role "讲解员" "https://s.coze.cn/image/xxx/" "正面" \
  --voice "女声，专业清亮，语速明快" \
  --tts \
  --voice-name "温柔女声 - vivi"

# 电商带货主播
python gen-video/scripts/update_material_files.py "./素材库.json" \
  --role "主播" "https://s.coze.cn/image/xxx/" "正面" \
  --voice "女声，活泼热情，语速快" \
  --tts \
  --voice-name "咪仔"

# 旁白（管线 B）
python gen-video/scripts/update_material_files.py "./素材库.json" \
  --narrator "https://s.coze.cn/image/kgWvY_NvVcQ/" \
  --voice "男声，稳重专业" \
  --tts \
  --voice-name "大壹"
```

### V5 阶段：写分镜

**台词格式**（管线 B 临时占位）：
```
角色名说："台词"
```

**⚠️ 管线 B 在 V5 阶段仍使用占位格式，V5.5 会替换为音频引用。**

### V5.5 阶段：TTS 注入（关键步骤）

**核心任务**：调用 `text_to_speech` 生成音频，注入分镜。

#### Step 1：收集所有台词

从 `.cvs` 文件中提取所有台词和对应的角色名。

#### Step 2：调用 text_to_speech

**示例**：
```python
text_to_speech(
    sentences=[
        {
            "text": "大家好，欢迎来到我们的科普频道。",
            "voice_name": "温柔女声 - vivi",
            "voice_style": "专业清亮，语速明快"
        },
        {
            "text": "今天我们要聊聊一个有趣的话题。",
            "voice_name": "温柔女声 - vivi",
            "voice_style": "专业清亮，语速明快"
        }
    ]
)
```

**返回**：音频 URL 列表

#### Step 3：音频入库

将生成的音频 URL 入库到素材库：

```bash
python gen-video/scripts/update_material_files.py "./素材库.json" \
  --role "讲解员" "音频URL" "台词1音频"
```

#### Step 4：注入分镜 resource_inputs

修改 `.cvs` 文件，将音频注入到对应 shot 的 `resource_inputs`：

**修改前**：
```json
{
  "shot_id": 1,
  "resource_inputs": [
    { "media_type": 2, "value": "https://场景图URL", "key": "场景名", "role": "reference_image" }
  ],
  "aigc_prompt": "讲解员说：\"大家好，欢迎来到我们的科普频道。\"不添加字幕，不添加BGM"
}
```

**修改后**：
```json
{
  "shot_id": 1,
  "resource_inputs": [
    { "media_type": 2, "value": "https://场景图URL", "key": "场景名", "role": "reference_image" },
    { "media_type": 1, "value": "https://音频URL", "key": "讲解员_台词1", "role": "reference_audio" }
  ],
  "aigc_prompt": "讲解员使用[音频1]的音色说：\"大家好，欢迎来到我们的科普频道。\"不添加字幕，不添加BGM"
}
```

**关键变化**：
1. `resource_inputs` 新增音频元素（`media_type: 1`, `role: reference_audio`）
2. `aigc_prompt` 中台词格式改为 `角色名使用[音频N]的音色说："台词"`

#### Step 5：校验

重新执行 V5.1 校验：
```bash
python gen-video/scripts/validate_video_script.py "./xxx_视频脚本.cvs" --material-library "./素材库.json" --auto-fix
```

## ⚠️ 五、管线选择常见错误

### 错误1：漫剧场景使用管线 B

❌ **错误做法**：
```bash
python gen-video/scripts/update_material_files.py "./素材库.json" \
  --role "陈念" "https://url/" "正面" \
  --voice "男声，青年" \
  --tts \
  --voice-name "林潇"
```

✅ **正确做法**：
```bash
python gen-video/scripts/update_material_files.py "./素材库.json" \
  --role "陈念" "https://url/" "正面" \
  --voice "男声，青年音色，温和清亮，语速中等，带有书卷气"
```

**原因**：漫剧角色音色多样，管线 A 的描述性音色更灵活。

---

### 错误2：数字人口播使用管线 A

❌ **错误做法**：
```bash
python gen-video/scripts/update_material_files.py "./素材库.json" \
  --role "讲解员" "https://url/" "正面" \
  --voice "女声，专业清亮"
```

✅ **正确做法**：
```bash
python gen-video/scripts/update_material_files.py "./素材库.json" \
  --role "讲解员" "https://url/" "正面" \
  --voice "女声，专业清亮" \
  --tts \
  --voice-name "温柔女声 - vivi"
```

**原因**：数字人口播需要口型同步，必须预生成音频（管线 B）。

---

### 错误3：管线 B 跳过 V5.5

❌ **错误做法**：
- V3 阶段加了 `--tts` 和 `--voice-name`
- V5 阶段写分镜后直接 V5.1 校验
- V7 直接生成视频

**结果**：V7 生成时没有音频，视频无声。

✅ **正确做法**：
- V3 阶段：入库时加 `--tts` 和 `--voice-name`
- V5 阶段：写分镜（占位格式）
- V5.5 阶段：TTS 生成音频 + 注入分镜（**必须执行**）
- V5.1 阶段：校验
- V7 阶段：生成视频

---

### 错误4：管线 A 手动展开音色公式

❌ **错误做法**：
```
aigc_prompt: "陈念是男声，青年音色，温和清亮，语速中等，带有书卷气，说：\"这玉佩，似乎与我有缘。\""
```

✅ **正确做法**：
```
aigc_prompt: "陈念说：\"这玉佩，似乎与我有缘。\""
```

**原因**：V5.1 auto-fix 会自动展开，手动展开容易出错且冗余。

## 📋 六、音色管线检查清单

| # | 检查项 |
|---|--------|
| 1 | 根据场景类型选择正确的管线（A 或 B） |
| 2 | 管线 A：V3 入库时只传 `--voice`，不加 `--tts` |
| 3 | 管线 B：V3 入库时传 `--voice` + `--tts` + `--voice-name` |
| 4 | V5 台词格式统一为 `角色名说："台词"` |
| 5 | 管线 B：V5.5 必须执行 TTS 注入 |
| 6 | 管线 B：音频注入后 resource_inputs 包含 `reference_audio` 元素 |
| 7 | 管线 B：aigc_prompt 中台词格式改为 `角色名使用[音频N]的音色说："台词"` |
