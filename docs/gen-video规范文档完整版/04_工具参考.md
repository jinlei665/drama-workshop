# 工具参考

> **所属 Skill**: gen-video
> **核心用途**: 所有 Python 脚本的参数速查，一张表搞定调用格式
> **何时读取**: V2-V5 阶段需要调用脚本时
> **前置理解**: 素材库是上游资产仓库，分镜表是下游工程文件。详见 `video-script-overview.md §0`

---

## ⚠️ 路径规则（必读）

**工作目录**：当前目录（用户文件所在目录）

**Skill 安装位置**：`./gen-video/`

**脚本调用格式**：
```bash
python gen-video/scripts/xxx.py "./素材库.json"
```

**示例**：
```bash
# 查询素材
python gen-video/scripts/update_material_files.py "./素材库.json" --list

# 校验分镜表
python gen-video/scripts/validate_video_script.py "./剧名_视频脚本.cvs" --material-library "./素材库.json" --auto-fix

# 多集项目
python gen-video/scripts/validate_video_script.py "./剧名_第1集_视频脚本.cvs" --material-library "./素材库.json" --auto-fix
```

**常见错误**：
- ❌ `python scripts/xxx.py` → 缺少 `gen-video/` 前缀
- ❌ `python /workspace/projects/gen-video/scripts/xxx.py` → 错误的绝对路径

## ⛔ 平台内置工具（非 Python 脚本，直接调用工具函数）

以下工具是平台内置的，**不是 Python 脚本**，严禁用 `bash_tool` 或 `python` 命令调用：

| 工具名 | 用途 | 使用阶段 |
|--------|------|---------| 
| `init_video_script` | 初始化 .cvs 文件 | V1 |
| `batch_gen_video_shots` | 批量生成视频 | V7 |

**⛔ 平台工具路径规则**：传参时**只写文件名**，不带任何路径前缀（注意：这条只针对平台内置工具，上面的 Python 脚本仍然用 `"./素材库.json"`）。
- ✅ `"video_script_file": "梦核_视频脚本.cvs"`
- ❌ `"video_script_file": "/app/data/files/梦核_视频脚本.cvs"`（绝对路径导致产物路径错乱）
- ❌ `"video_script_file": "./梦核_视频脚本.cvs"`（`./` 前缀对平台工具可能有问题）

**⛔ 不存在以下脚本**（Agent 常虚构的名字）：
- ~~`append_shots.py`~~ → 追加分镜用 Python 代码操作 `data["shot_scripts"]`
- ~~`register_material.py`~~ / ~~`asset_manager.py`~~ → 素材入库只能用 `update_material_files.py`
- ~~`batch_gen_video_shots.py`~~ → 这是平台内置工具，直接调用工具函数
- ~~`compose_tracks.py`~~ → 编辑 project_tracks 用 Python 代码操作 `data["project_tracks"]`

**⛔ 素材库只能通过脚本入库，严禁用 edit_file 手写 JSON**——手写极易写错嵌套结构（如 `image_details` 应为 `role_details`）。

## V8.5 轨道编辑（Python 范式）

编辑 project_tracks 时用 `bash_tool` 执行 Python 代码。**详细参考 `track-composition-guide.md`**。

标准入口：
```python
import json

with open('xxx_视频脚本.cvs', 'r', encoding='utf-8') as f:
    data = json.load(f)

# ⛔ 字段名是 project_tracks（不是 tracks）
tracks = data.get('project_tracks') or []
# tracks[0]=主视频轨, tracks[1]=叠加轨, tracks[2]=文字轨, tracks[3]=BGM轨, tracks[4]=音频轨

# ... 编辑操作（trim/speed/transform 等，见 track-composition-guide.md）...

data['project_tracks'] = tracks
with open('xxx_视频脚本.cvs', 'w', encoding='utf-8') as f:
    json.dump(data, f, ensure_ascii=False, indent=2)
```

### analyze_bgm.py（V2.5，仅 MV/卡点/混剪场景）

| 参数 | 类型 | 说明 |
|------|------|------|
| `--bpm` | 可选 | 已知 BPM（快速模式，从 select_music 的 Extra.beat_info.bpm 获取） |
| `--duration-ms` | 可选 | 音乐时长毫秒（快速模式必填，select_music 的 Duration×1000 或 gen_music 的 duration_ms） |
| `--audio-url` | 可选 | 音频 URL（完整分析模式，需要 librosa） |
| `--output` / `-o` | 可选 | 输出文件路径（默认 beat_map.json） |

两种模式：
- **快速版**（select_music 已知 BPM）：`python gen-video/scripts/analyze_bgm.py --bpm 128 --duration-ms 60000`
- **完整版**（gen_music 生成的音频）：`python gen-video/scripts/analyze_bgm.py --audio-url "https://s.coze.cn/audio/xxx/"`

输出 `beat_map.json`，供 V5 写分镜时作为节拍参考。

### update_material_files.py（V3/V4）

**⛔ 用素材名称操作，不要手动管理 material_id（ID 由脚本自动生成，平台可能覆盖）**

| 参数 | 类型 | 说明 |
|------|------|------|
| `素材库.json` | 位置参数 | 素材库路径（不存在时自动创建） |
| `--list` | 可选 | 列出已有素材（V3 前必须执行） |
| `--role "名称" "URL" "备注"` | 添加角色 | 如 `--role "诸葛亮" "https://url" "正面"` |
| `--scene "名称" "URL" "备注"` | 添加场景 | 如 `--scene "东吴军帐" "https://url" "全景"` |
| `--item "名称" "URL" "备注"` | 添加道具 | 如 `--item "羽扇" "https://url" "特写"` |
| `--narrator "URL"` | 添加旁白 | 占位图 URL，名称固定为"旁白" |
| `--video "名称" "URL"` | 添加用户视频 | 模式 3（剪辑）用户视频入库 |
| `--voice "描述"` | 可选 | 角色/旁白音色描述 |
| `--tts` | 可选 | 管线 B（预制音色）：加此标记后 validate 不展开音色，留给 V5.5 TTS 注入 |
| `--replace` | 可选 | 替换已有文件（默认追加） |

同名素材自动追加文件（如先加正面、再加侧面）。material_id 由脚本自动分配，**agent 无需关心 ID**。

**管线决策**：scene-rule 标注 🅰️ 管线 A → 只传 `--voice`；标注 🅱️ 管线 B → 传 `--voice` + `--tts`。

示例：
- `python gen-video/scripts/update_material_files.py "./素材库.json" --list` ← **V3 前必查**
- `python gen-video/scripts/update_material_files.py "./素材库.json" --role "男主" "https://url" "正面" --voice "男声，青年音色..."` ← 管线 A（漫剧/氛围片/预告片）
- `python gen-video/scripts/update_material_files.py "./素材库.json" --role "讲解员" "https://url" "正面" --voice "女声，专业清亮..." --tts` ← 管线 B（数字人/科普/电商）
- `python gen-video/scripts/update_material_files.py "./素材库.json" --scene "雪山" "https://url" "全景"`
- `python gen-video/scripts/update_material_files.py "./素材库.json" --narrator "https://s.coze.cn/image/kgWvY_NvVcQ/" --voice "男声，老年音色..."`
- `python gen-video/scripts/update_material_files.py "./素材库.json" --video "用户上传视频" "https://www.coze.cn/s/xxx/"` ← 模式 3 用户视频入库

⛔ 常见错误：
- ~~手动指定 material_id~~ → 用 `--role/--scene/--item` 替代，ID 自动生成
- ~~手动构造完整素材库 JSON~~ → 禁止
- ~~`--tag 3`~~ / ~~`--tag_category 1`~~ → 无此参数

### edit_tracks.py（V8.5 / E-Flow 剪辑）

| 参数 | 类型 | 说明 |
|------|------|------|
| `脚本.cvs` | 位置参数 | .cvs 或 .cvp 文件路径 |
| `--status` | 可选 | 查看轨道状态（视频/字幕/BGM 的时间分布） |
| `--speed <倍率>` | 可选 | 全片变速（如 1.25 = 加速 25%）。自动处理 trim+speed+TargetTime+字幕同步 |
| `--trim <shot_id> <start_ms> <end_ms>` | 可选 | 裁剪指定 shot，保留素材内 start-end 范围。后续元素 TargetTime 自动前移 |
| `--remove-shot <shot_id>` | 可选 | 删除指定 shot，后续自动前移 |
| `--output` / `-o` | 可选 | 输出路径（默认覆盖原文件） |

示例：
- `python gen-video/scripts/edit_tracks.py "./xxx.cvs" --status` ← 查看轨道
- `python gen-video/scripts/edit_tracks.py "./xxx.cvs" --speed 1.25` ← 全片加速 25%
- `python gen-video/scripts/edit_tracks.py "./xxx.cvs" --trim 1 0 9500` ← Shot 1 只保留前 9.5s
- `python gen-video/scripts/edit_tracks.py "./xxx.cvs" --remove-shot 3` ← 删除 Shot 3

**⛔ 操作完 finish .cvs，不需要重调 create_video_project。**

### build_tracks.py（V8.5 模式 3 用户视频剪辑）

| 参数 | 类型 | 说明 |
|------|------|------|
| `脚本.cvs` | 位置参数 | .cvs 文件路径 |
| `--video-url <URL>` | 可选 | 用户视频的 CDN URL（不是 Coze 分享链接） |
| `--duration-ms <毫秒>` | 可选 | 原视频总时长（与 --video-url 配合） |
| `--cuts '<JSON数组>'` | 必填 | 裁剪方案，每段：`{"start_ms": 起始, "end_ms": 结束, "label": "字幕"}` |

示例：
```bash
python gen-video/scripts/build_tracks.py "./xxx.cvs" \
  --video-url "https://cdn.example.com/video.mp4" \
  --duration-ms 65000 \
  --cuts '[{"start_ms":0,"end_ms":3000,"label":"开场"},{"start_ms":5000,"end_ms":9000,"label":"答案"},{"start_ms":18000,"end_ms":28000,"label":"核心内容"}]'
```

**⛔ 必须在 create_video_project 之前执行**（tracks 写入 .cvs 后再调 V8）。
